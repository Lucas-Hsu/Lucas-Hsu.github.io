Update On My Old Games

Small Fixes and Extra Builds

Hello Visitor,

In this blog post, I will describe some recent changes I made to the games I developed over the past few years. At the end of the post, I will also talk about how I’m updating Frabjous Fuzzk from the Beta to the Official Release.

All of these games are currently only accessible on my Itch.io page, which is linked on the EXTERNAL LINKS page. If you’re here in the future, you will find individual pages for them in the MY GAMES page.

Old Games

In general, I made as many builds for these old games as my game engine and engine version supported. All games are now playable in the browser in some form or another. Other than that, certain games got some simple quality-of-life improvements.

BounceGame

BounceGame (Godot) now has a new Linux version (x86_64) in addition to the pre-existing builds. Not many changes were needed as I already polished it when I recovered the PyGame version of BounceGame last year.

Merge Game

Merge Game (Unity) now has separate builds for Windows (64-bit / 32-bit), macOS (Intel / M-Series / Universal), and Linux (x86_64), as well as a new HTML5 version. The most notable change, however, is the addition of a simple background music track. If you unfortunately don’t find it calming as I do, you can mute the music by pressing [M].

Spray Game

Spray Game (Unity) now has the same new builds like with Merge Game, though it already has a web version. This update merged the Baby Mode into the base game, so that it is now just a difficulty toggle. The SprayGame Manual has been updated accordingly with an explanation of that difficulty setting.

Parody: 1999

Parody: 1999 (Unity) also got the new builds. Sadly, the HTML5 version does not support Unity VideoClips, so all the cutscenes fail to play. The cutscenes are a crucial part of the Parody: 1999 experience! You’ll just have to watch the cutscenes separately through the combined video of all cutscenes I uploaded.

PUSH

PUSH (Godot) has a few new builds: Windows (64-bit / 32-bit / arm64), Apple (Universal), and Linux (x86_64, x86_32, arm64, and arm32). PUSH is quite an unoptimized game, so it really needs any performance boost it can get. The HTML5 version exists too, but due to unknown issues (maybe because Vulkan doesn’t work on the web), rendering is broken and ugly.

Frabjous Fuzzk

I’ve finally resumed development for Frabjous Fuzzk. All I’m doing before the official release is going through the long list of QoL improvements feedback I received from my family, friends, and teachers. For example, the player character will now grow bigger as you level up, increasing your max stamina as well as your kick range. The central theme of this update is Cooperation, so I’m increasing the helpfulness of your Fuzzk friends in more scenarios. They can now protect you from Toe stings! Apart from that, I’m just doing a couple of art updates, like the world borders and Shells.

As you might have noticed in the teaser render above, I now have a 3D model for the Cyan Fuzzk! By simply changing the material colors, I now have a model for all 7 Fuzzks. The buildings are simple billboards, the map is a selectively elevated subdivided plane, and the background is a hand-drawn image. I will be using them to render the intro cutscene in which Red Fuzzk gets captured and the ending cutscene in which we defeat the Scary Monster. The 3D models won’t be in the actual game.

You might remember—as I certainly do—I planned numerous extra features for Frabjous Fuzzk, such as the dialogue system between Fuzzks, the ability of Fuzzks to automatically fight nearby enemies, or the addition of an extra prey/predator creature in the Overworld. They would undoubtedly be great additions to this game. However, they won’t be added. The game’s code is a tangled mess due to my use of a bunch of “quick and dirty” fixes when I tried to ship the game within two weeks last summer. It’s a shame because I tried to design the game to be modular and adaptive, but I obviously failed, considering it gives me a headache whenever I’m adding anything new to the game. Lesson learned. In the design phase of my next game, I’ll be sure to outline the programming systems too, not just the story and game mechanics.

Still, I’m having a lot of fun updating Frabjous Fuzzk, which means I’ll be working on the game quite consistently. The full version of Frabjous Fuzzk will be out by next week. Stay tuned!

All the best,
